# implementation-plan2：天天象棋网页版 V1.1 对标补强计划

## Summary

在现有五阶段可玩版本基础上，做一轮“对标微信天天象棋小程序”的体验补强。目标不是重做，而是把当前版本从“流程可玩”推进到“更像、更顺、更完整”：补齐大厅入口密度、局内菜单、结算复盘细节、0 手复盘处理、首页视觉挤压、揭棋/翻翻棋最小玩法、残局辅助功能和移动端细节。

继续保持纯前端实现：React + TypeScript + Vite，不接真实后端、不接微信登录、不做真钱/充值/广告/真实社交/真实在线匹配，不复制天天象棋原版素材。

## Phase 1：核心体验缺口修复

- 修复首页首屏视觉挤压：右侧预览棋盘改为更小的活动挂件/棋局预览卡，保证桌面 1280x720 下不截断，手机竖屏下首屏优先展示 banner、入口和信息流。
- 修复结果页 0 手复盘问题：当 `result.moves.length === 0` 时，禁用“复盘分析”按钮，显示“暂无可复盘棋谱”；复盘页仍保留兜底空状态。
- 统一路由标题逻辑：左侧模式、底部 tab、对局、结算、复盘的顶部标题必须和当前内容一致。
- 增加全局 toast：用于低步时限制、未开放入口、复制/分享占位、悔棋/提和占位提示。
- 验收标准：试玩不会出现“按钮可点但无反馈”、标题错乱、棋盘截断、0 手空复盘误导。

## Phase 2：象棋大厅对标扩展

- 将象棋大厅从当前“棋力评测/普通对局”扩展为小程序式入口列表：
  - 棋力认证赛：可进入匹配，影响本地积分。
  - 棋力评测：可进入匹配，显示 15 分钟局时、前 3 回合 30 秒、之后 90 秒。
  - 训练场：可进入人机对局，不影响积分。
  - 华山论剑：显示活动场入口，点击 toast“活动暂未开放”。
  - 5 分钟场、10 分钟场、20 分钟场：复用对局流程，但展示不同局时配置；先只调整展示与初始总时长。
  - 好友对战：显示房间制入口，点击 toast“好友房间后续开放”。
- 大厅布局改成更像小程序的多行对手/场次卡：左侧模式入口，中间 VS/人数/准入资源，右侧入场说明。
- 匹配弹窗保留现有动画，但文案根据入口动态变化。
- 验收标准：象棋模块入口密度接近微信小程序，不再显得过空；至少 3 类入口可进入真实对局，其余都有明确占位反馈。

## Phase 3：局内菜单、设置和聊天补强

- 将局内底部工具栏调整为：
  - 左下“主页/菜单”按钮：点击展开竖向菜单。
  - 菜单项：离开、认输、提和(3)、悔棋(3)、设置。
  - 聊天按钮保留在底部，打开现有聊天抽屉。
  - 右侧增加“邀请旁观”静态按钮，点击 toast“本地版暂不支持旁观”。
- 菜单行为：
  - 离开：二次确认，确认后按认输结算。
  - 认输：二次确认，确认后进入结算。
  - 提和(3)：次数展示，点击后 toast“本地人机暂不支持提和”，次数不扣。
  - 悔棋(3)：如果至少有一组双方走子，则回退最近一轮红黑走法并扣 1 次；否则 toast“暂无可悔棋步数”。
  - 设置：打开设置弹窗，暂停计时，关闭后恢复。
- 设置补强：保留现有“显示走子提示”，新增“棋盘坐标线”“吃子/将军动画”“背景音乐”“下棋音效”等开关；除走子提示外，其余先作为 UI 状态，不影响核心规则。
- 聊天补强：魔法表情点击后写入输入框；发送后切到聊天记录，追加“我方：xxx”；45 字限制保持。
- 验收标准：局内操作路径更接近天天象棋；菜单项均有明确效果或占位反馈；计时暂停/恢复稳定。

## Phase 4：结算与复盘精修

- 结算页补齐小程序信息密度：
  - 双方头像、昵称、段级。
  - 胜负标题、结果原因、回合数、用时、日期。
  - 积分变化、本局积分、段位进度条。
  - 小棋盘缩略图和最近一步标记。
  - 按钮：切换对手、再来一局、分享、复盘分析。
- 分享按钮不做真实分享，点击 toast“分享功能为本地占位”。
- 复盘页补强：
  - 左侧大棋盘，右侧棋谱信息面板。
  - 信息面板显示：棋谱标题、红方、黑方、结果、日期、评论占位、注释占位。
  - 控制按钮：上变、上一步、播放/暂停、下一步、下变；上变/下变先置灰或 toast“暂无变着”。
  - 底部栏保留：菜单、棋谱信息、上一步、下一步、分析；分析点击 toast“分析功能后续接入”。
- 验收标准：结算页不再像简单结果卡；复盘页即使没有 AI 分析，也有天天象棋式棋谱面板和完整控制区。

## Phase 5：残局、揭棋、翻翻棋最小玩法

- 残局增强：
  - 增加步数、计时、重来、提示、评论入口占位。
  - 提示按钮高亮推荐红车落点；评论入口打开“棋谱评论”弹窗，包含题名、参与人数、过关率、评论输入。
  - 保留当前一手制胜玩法。
- 揭棋最小可玩：
  - 新增 `JieqiPiece`：包含真实棋子信息、是否暗子、是否已揭示。
  - 初始多数棋子显示“暗”，帅/将等少量明示。
  - 选择暗子时仍按其真实棋种生成合法落点；首次移动后翻开显示真实棋种。
  - 对手仍用现有 AI 简化应手，移动暗子后也揭示。
  - 揭棋结算复用象棋结算，但文案显示“揭棋评测”。
- 翻翻棋最小可玩：
  - 新增 4 列暗子棋盘，点击暗子翻开预设棋种。
  - 展示双方头像、资产、血量条、当前倍率。
  - 不实现完整吃子胜负，只做翻开、倍率展示、规则弹窗和强退确认占位。
- 验收标准：残局有“每日残局”味道；揭棋不再只是静态预览；翻翻棋至少可翻子并展示规则/倍率/血量。

## Phase 6：视觉、移动端与验收收尾

- 桌面适配：
  - 1280x720 下首页、大厅、对局、结算、复盘不出现关键内容截断。
  - 棋盘居中，玩家计时区不挤压棋盘。
- 手机竖屏适配：
  - 390x844 下左侧玩法导航横向滚动可见。
  - 棋盘、底部 tab、弹窗、聊天抽屉不横向溢出。
  - 结果页和复盘页可滚动，但顶部标题和关键按钮不丢失。
- 视觉统一：
  - 弹窗统一浅米色面板、木质/红色标题和圆角按钮。
  - 禁用按钮有明显灰态。
  - toast 统一位置、颜色、出现时长。
- 最终验收截图：
  - 首页、象棋大厅、本场说明、对局、局内菜单、聊天、设置、结算、复盘、残局、揭棋、翻翻棋、手机首页。
- 验收标准：核心路径连续试玩 3 次无阻断；所有未开放功能都有 toast 或灰态；测试和构建通过。

## Interfaces / Types

- 在现有前端内部扩展类型，不新增 HTTP API：
  - `LobbyMode`：大厅入口配置，包括标题、局时、步时、消耗、奖励、是否可进入对局。
  - `ToastState`：toast 文案和类型。
  - `GameMenuAction`：离开、认输、提和、悔棋、设置。
  - `GameSettings`：扩展音效、消息、走子提示、坐标线、动画等开关。
  - `JieqiPiece`：暗子状态、真实棋种、是否已揭示。
  - `FlipPiece`：翻翻棋暗子、阵营、棋种、是否翻开。
- `GameResult` 增加本地展示字段：
  - `endedAt`
  - `pointsDelta`
  - `rounds`
  - `modeLabel`
  - `scoreProgress`
- `ChessBoard` 增加可选展示配置：
  - `compact`
  - `disabled`
  - `showCoordinates`
  - `hiddenPieceIds`
  - `pieceLabelOverride`
- 主要实现集中在 `src/App.tsx`、`src/game.ts`、`src/styles.css`；如文件继续膨胀，可在实施时拆出 `src/ui-data.ts` 存放大厅模式、弹窗文案、静态配置。

## Test Plan

- 单元测试：
  - 现有象棋规则测试全部保留。
  - 新增悔棋：无步数不可悔、有红黑一轮可回退、次数扣减。
  - 新增揭棋：暗子首次移动后揭示，已揭示棋子保持真实标签。
  - 新增复盘：0 手结果禁用复盘入口，有棋谱时可步进。
- 流程测试：
  - 首页 → 象棋大厅 → 棋力认证赛 → 匹配 → 开始 → 走子 → AI 应手 → 认输 → 结算 → 复盘。
  - 首页 → 排位赛 → 10 分钟场 → 匹配 → 退出确认 → 结算。
  - 残局 → 提示 → 走正确解 → 成功 → 重来。
  - 揭棋 → 选择暗子 → 移动 → 揭示。
  - 更多玩法 → 翻翻棋 → 翻子 → 规则弹窗 → 强退确认。
- 视觉测试：
  - 桌面 1280x720、手机 390x844 截图检查 12 个关键页面。
  - 检查无横向溢出、文字不压按钮、弹窗不遮住关键操作。
- 构建测试：
  - `tsc -p tsconfig.json`
  - Vite production build
  - 规则测试脚本

## Assumptions

- 本轮只做本地可玩和视觉/交互对标，不做后端、账号、联网、排行榜、真实好友房间、真实分享、支付、广告。
- 所有“未开放”功能必须有灰态或 toast，不能静默无反应。
- 不使用天天象棋原版图片、商标或专有素材，只延续木纹、棋盘、圆形棋子等通用风格。
- 优先级固定为：核心象棋体验补强 > 结算复盘 > 大厅/首页对标 > 残局/揭棋/翻翻棋最小玩法 > 视觉收尾。
